「また新しいAIツールが出た...」

[Cursor](https://www.cursor.com/)、[GitHub Copilot](https://github.com/features/copilot)、[Claude Code](https://docs.anthropic.com/en/docs/claude-code/overview)、[Windsurf](https://codeium.com/windsurf)、[Cline](https://github.com/cline/cline)、[Devin](https://devin.ai/)...。2024年から2026年にかけて、AIコーディングツールは爆発的に増えた。毎週のようにXのタイムラインに「○○使ってみた」「△△がヤバい」という投稿が流れてくる。

「全部試さないと置いていかれる」という焦り、わかる。僕もそうだった。

でも正直、**疲れてない？**

:::message
この記事では、AIツール乱立時代に疲弊しないための「選ばない技術」について書く。全部追わなくても大丈夫。**自分の軸を持てば、2026年は生き残れる。**
:::

## AIツール疲れの正体

なぜ僕らは疲れるのか。原因は3つある。

### 1. 選択肢の多さ

AIコーディングツールだけでも、主要なものを挙げるとこうなる：

- [Claude Code](https://docs.anthropic.com/en/docs/claude-code/overview)
- [GitHub Copilot](https://github.com/features/copilot)
- [Cursor](https://www.cursor.com/)
- [Windsurf](https://codeium.com/windsurf)
- [Cline](https://github.com/cline/cline)
- [Devin](https://devin.ai/)
- [Amazon Q Developer](https://aws.amazon.com/q/developer/)
- [JetBrains AI Assistant](https://www.jetbrains.com/ai/)

これに加えて、Claude Code Skillsだけでも[awesome-claude-skills](https://github.com/travisvn/awesome-claude-skills)リポジトリには大量のスキルが公開されている。「結局どれを入れればいいの？」という状態だ。

比較検討するだけで時間が溶ける。

### 2. 変化の速さ

> 「AIツールは長期契約しない方が良い」

これは実際に色々試した人の教訓だ。数ヶ月で主力ツールが変わることは珍しくない。2024年に「これだ！」と思ったツールが、2026年には別のツールに置き換わっている。

年額プランは魅力的だけど、1年後もそのツールが最前線で使えるか、誰にも分からない。

### 3. FOMO（Fear of Missing Out）

「使わないと損する」「置いていかれる」という恐怖。

XやZennで「○○で生産性3倍になった」という記事を見ると、自分も試さないといけない気がしてくる。でも、その記事を書いた人と自分のユースケースは違う。

**全員に効くツールなんてない。**

## 「選ばない」という選択肢

ここで提案したいのが「選ばない」という戦略だ。

誤解しないでほしいのは、「AIツールを使うな」という話ではない。「全部試すのをやめる」「今は判断を保留する」という意味だ。

情報収集と実際の導入を分けて考える。

僕は野球の「1軍・2軍・ベンチ」というメタファーで整理している：

| ランク | 役割 | アクション |
| --- | --- | --- |
| **1軍** | 今メインで使っているツール | 深く使い込む |
| **2軍** | サブで使っている、または組織標準のツール | 限定的に活用 |
| **ベンチ** | 存在は知っているが、今は様子見 | 情報収集のみ |

新しいツールが出たら、まずベンチに入れる。すぐに1軍にしない。3ヶ月くらい業界の評判を見て、自分のユースケースに合いそうなら2軍で試す。

## 用途別・3層フレームワーク

「じゃあ何を基準に選ぶの？」という疑問に答えるため、3層のフレームワークを紹介する。

:::message
**3層フレームワーク**
1. **機能層** — 何ができるか
2. **運用層** — 継続して使えるか
3. **コスト層** — 見えないコストは何か
:::

### 機能層: 何ができるか

最初に見るのは機能だ。でも「機能が多い = 良い」ではない。

**評価ポイント:**
- 自分のユースケースの80%をカバーするか
- 得意な領域は何か（コード生成？レビュー？リファクタリング？）
- 自分の技術スタックに対応しているか

Claude Code Skillsの選定でも同じ考え方が使える。[awesome-claude-skills](https://github.com/travisvn/awesome-claude-skills)には大量のスキルがあるけど、全部入れる必要はない。自分の業務で頻繁に使う用途に絞って3〜5個選ぶ方が、コンテキストを圧迫せずに済む。

### 運用層: 継続して使えるか

導入して終わりではない。継続して使えるかが重要だ。

**評価ポイント:**
- 学習コストはどれくらいか
- チームの最も遅い人が使えるか（組織導入の場合）
- セキュリティ要件を満たすか
- 障害時の影響範囲は？

SREの視点を持つと、「開発効率が上がった」だけでは評価が不十分だとわかる。運用コストやセキュリティリスクを含めて考える必要がある。

### コスト層: 見えないコストは何か

月額料金だけがコストではない。

**評価ポイント:**
- 月額/年額の料金
- API利用料（従量課金の場合）
- 学習にかかる時間
- 切り替えコスト（他ツールに移行する場合の痛み）

:::message alert
「無料だから試してみよう」は危険だ。時間もコストだから。学習に10時間かけて、結局使わなかったら、それは10時間の損失だ。
:::

## 実践例: 僕の2026年スタメン構成

具体例があった方がイメージしやすいと思うので、僕の現在の構成を公開する。

### 1軍: Claude Code

メインで使っているのは[Claude Code](https://docs.anthropic.com/en/docs/claude-code/overview)。理由は：

| 評価層 | 評価 |
| --- | --- |
| **機能層** | コード生成、レビュー、リファクタリングすべてで高品質 |
| **運用層** | CLIベースで既存のワークフローに組み込みやすい |
| **コスト層** | MAXプランで定額、APIコストを気にしなくていい |

Skillsは以下の4つに絞っている：
- セキュリティレビュー用
- テスト生成用
- ドキュメント生成用
- コードレビュー用

[awesome-claude-skills](https://github.com/travisvn/awesome-claude-skills)には魅力的なスキルがたくさんあるけど、入れすぎるとコンテキストを圧迫する。本当に頻繁に使うものだけに絞った。

### 2軍: [GitHub Copilot](https://github.com/features/copilot)

チームの標準ツールとして使っている。理由は：

| 評価層 | 評価 |
| --- | --- |
| **機能層** | タブ補完が優秀、思考を妨げない |
| **運用層** | VS Code統合が安定している、チーム全員が使える |
| **コスト層** | 組織契約で個人負担なし |

### ベンチ: [Cursor](https://www.cursor.com/)、[Windsurf](https://codeium.com/windsurf)、[Cline](https://github.com/cline/cline)

存在は知っているし、評判も良いのは分かっている。でも今は様子見。

理由は「**今の構成で困っていないから**」。Claude CodeとCopilotで僕のユースケースの95%はカバーできている。新しいツールを試す時間があるなら、今使っているツールを深掘りする方が効率的だ。

## 疲れないための3つの習慣

最後に、AIツール疲れを防ぐための習慣を3つ紹介する。

### 1. 週1回だけ情報収集する日を決める

毎日Xやはてブを巡回して「新しいツールが出た」と反応していたら疲れる。

僕は日曜の朝に「週刊AI駆動開発」的なまとめ記事だけ読むようにしている。平日は意図的に情報を遮断する。

### 2. 「試す」と「導入する」を分ける

新しいツールを試すときは、本番プロジェクトではなくサンドボックス環境で。

「ちょっと試してみよう」で本番環境をいじると、戻れなくなる。試す環境と導入する環境は明確に分ける。

### 3. 3ヶ月ルール

新しいツールが出ても、3ヶ月は様子見する。

初期の熱狂が落ち着いた後に、冷静な評価記事が出てくる。「使ってみたけど微妙だった」という記事は、リリース直後には出てこない。3ヶ月待てば、本当の評判が見えてくる。

## まとめ: 2026年を生き残るために

AIツールに疲れているなら、それは真剣に向き合っている証拠だ。

でも、**全部追わなくていい。**

:::message
**この記事のポイント**
- 3層フレームワーク（機能・運用・コスト）でツールを評価する
- 1軍・2軍・ベンチで整理し、新しいツールはまずベンチに入れる
- 3ヶ月ルールで初期の熱狂に流されない
- 大事なのは「自分の軸を持つこと」
:::

2026年、AIの進化は止まらない。でも、僕らが疲弊する必要はない。
